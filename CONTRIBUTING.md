# Contributing to Mirai

Thank you for your interest in contributing. This document describes how to set up a development environment, run tests, and submit changes.

## Development setup

```bash
# After forking, replace <your-user> with your GitHub username:
git clone https://github.com/<your-user>/Mirai.git
cd Mirai
python -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -e ".[dev]"
```

Optional: install [pre-commit](https://pre-commit.com/) hooks after cloning:

```bash
pre-commit install
```

## Running tests

See [docs/TESTING.md](docs/TESTING.md) for context. From the repository root:

```bash
pytest
```

CI (see [.github/workflows/ci.yml](.github/workflows/ci.yml)) runs on each push and PR: Python (`compileall`, `ruff`, `pytest`), plus TypeScript, C++, Go, Swift, and Java SDK builds.

## Releases (PyPI & npm)

Maintainers can publish **after** bumping versions in [`pyproject.toml`](pyproject.toml) (Python) and [`mirai/sdk/typescript/package.json`](mirai/sdk/typescript/package.json) (TypeScript) to match the git tag.

Before pushing a release tag, run a local smoke check from a fresh `.[dev]` environment:

```bash
pytest
python -m build
```

1. Create an annotated tag: `git tag -a v0.1.0 -m "v0.1.0"` and `git push origin v0.1.0`.
2. Configure repository **Secrets**:
   - `PYPI_API_TOKEN` — [PyPI API token](https://pypi.org/help/#apitoken) for the `mirai-agent` project.
   - `NPM_TOKEN` — [npm automation token](https://docs.npmjs.com/creating-and-viewing-access-tokens) with publish rights for `mirai-sdk`.
3. The [Release workflow](.github/workflows/release.yml) runs on `v*` tags: builds the Python wheel/sdist and publishes to PyPI, and publishes the TypeScript package to npm. Jobs are skipped if the corresponding secret is missing.

Forks and contributors without these secrets will not run the publish steps.

## Code style

- Prefer matching existing patterns in the file you edit.
- Python 3.10+ type hints (`str | None` instead of `Optional[str]` where consistent).
- Run `ruff check mirai tests` and `ruff format mirai tests` if configured (see `.pre-commit-config.yaml`).

## Compatibility expectations

- Treat the `mirai` CLI, documented HTTP API, and generated SDK templates as the main user-facing surfaces.
- Avoid adding undocumented dependencies on internal modules such as `mirai.core.*` unless the change is clearly marked as internal-only.
- If a change affects a public workflow or contract, update `README.md`, `docs/HTTP_API.md`, and `CHANGELOG.md` in the same PR.

## What not to commit

Do **not** commit:

- `__pycache__/`, `*.pyc`, `*.pyo`
- `.venv/`, local IDE-only paths
- `*.egg-info/` (generated by editable installs)
- Build artifacts under `mirai/ui/.web/`, `mirai/sdk/**/node_modules/`, `mirai/sdk/cpp/build/`, etc. (see [.gitignore](.gitignore))

If you accidentally committed generated files, remove them from the index with `git rm --cached <path>` and ensure they are listed in `.gitignore`.

## Issues before pull requests

For **non-trivial** changes (new features, public API changes, refactors, or anything that needs design discussion), **open an issue first** and describe the problem or proposal. Link that issue from your pull request so maintainers can triage and avoid duplicate work.

Small, obvious fixes (typos, clear one-line bugs, documentation corrections) may go straight to a pull request without a prior issue.

## Pull requests

1. Fork the repository and create a branch from `main`.
2. Make focused changes; prefer one logical concern per PR when possible.
3. Add or update tests when you change behavior.
4. Describe **what** changed and **why** in the PR description.

## Community standards

Please read [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md).

## Security issues

Do **not** open a public issue for security vulnerabilities. See [SECURITY.md](SECURITY.md).

## SDKs (Go, Swift, Java, C++, TypeScript)

- **Go / Java / C++ / TypeScript** SDKs are built on Linux in CI.
- **Swift** SDK is built on a **macOS** runner (Package platforms include Apple OSes).
- **Unreal Engine (UE5)** modules require a local UE installation; CI does not compile UE5. See [mirai/sdk/README.md](mirai/sdk/README.md).
